# 第4章：流程的抉择

档案馆的线索指向城邦边缘一座废弃多年的工厂。

解耦在当天傍晚出了创世之城的西门。城墙外是五百米宽的开阔缓冲带——一片被平整压实的灰色硬质地面，在日光下反着一种不刺眼但持久的白光。缓冲带尽头是一条废弃了不知多久的运输通道，地面残留着两条平行的金属轨道，锈迹从轨道的两端向中间缓慢推进。通道终点就是工厂——一栋低矮的、几乎完全嵌入地表的长条形建筑。它的外墙被锈迹和干涸了不知多少年的有机沉积物覆盖着，从墙角到屋顶没有一块完整的原色区域。但建筑内部有灯光透出来——不是稳定的照明光源，是终端面板和传感器节点在待机状态下的指示灯光——从裂缝和气窗缝隙中透出几束微弱的蓝色和琥珀色光。建筑内的备用能源系统仍在运转。

工厂的大门是一道单扇推拉门，没有任何开锁装置——门由一组机械联动结构控制，推拉方向的位移末端连接着一个齿轮组，齿轮组的啮合面深嵌在地面以下的轨道中。解耦握住门把手的金属杆向下压了一段行程没有任何咬合阻力，然后听到了沉重的齿轮转动声。

门开了。他踏进大门的那一刻，身后的门自动复位了——不是被推回来的，是机械联动结构在门开到设定角度后自动释放了张力。他身后的通路在缓慢的重力作用下被重新闭合封闭。

建筑内部是一条狭窄的主通道，两侧墙壁上每隔十步嵌着一组小型传感器节点——待机灯在昏暗的光线中亮着，以大致均匀的间隔在走廊两侧排列开来。墙面是裸露的压制金属板，多处可以见到螺栓连接的安装轨迹。空气的温度比外面略低。他的脚步声在管道中每一步都触发了一次壁面声反射——在这条没有任何窗户或通风口的长直达三十余米的通道中，每一次振动沿着金属构件的传导路径被传感器网络捕捉，然后以结构化的状态读数在通道更深处的终端面板上汇总。

解耦走了十几步，前路被一道闭锁的闸门截断。闸门的金属面上用蚀刻工艺嵌入了一组格式规则——不是彩色字符，也不是投影，是直接在金属面上用腐蚀剂处理出的永久性文字。文字旁边是一个操作面板，面板上排列着一组条件判断的输入控件。

闸门上刻着一行字："选择一条可通过的真实路径。"

面板上排列着一个条件分支结构——一条主体判定线和对应的两条备选路径。解耦扫了一遍条件表达式和上下文的判定参数：一个温度值读数、一个负载值读数。当前环境的温度值低于阈值，负载值低于上限参数。只有if分支的条件同时满足两条参数的要求。

他按下了if分支对应的确认键。闸门的锁止结构传来一声金属制退器的释放声，然后闸门向上收起。

第二道闸门在十几步之外等着他——不需要条件判断。面板上排列着一组重复执行指令结构，以及一个嵌入式传感器读数工具。指引文字描述的要求：持续读取周围环境的传感器的输入浮动值——保持这个值在规定的区间内——直到读数自然收敛。

不是一次通过的单次判断——是一个需要持续监控直到法定范围稳定的过程。解耦把机械兽从外套内袋中取出，通过它的外围扫描模块取得了环境的当前精确读数——环境温度低于区间下限。不是信号故障或者传感器配置异常，是这个密闭空间的内在温度长期保持在目标温度以下。

他无法通过操作终端升高环境的整体温度。他必须找到另一个办法满足循环退出条件。他没有采用强行修改面板回传参数之类的规避方式——那样阅读器会对判定方法的内测试次数计数，过程终止后闸门不大会开放通过路径。

走廊两侧的墙壁上排列着几台设备，外壳上贴着冷却效率的额定标牌。他在其中一台的外壳侧面找到了一个可调节的百叶式散热门，对所有挡片的开度做了调节——机柜的强制散热排风温度在挡片调节后汇聚到闸门前的传感器位置上。面板上循环体的温度变量在接收了新的局部信号后一致链路终端从等待切换为退出，闸门升起。

他走过闸门时，经过的设备内部冷却风机发出了短暂的转速调整声。他没停。

第三道闸门——面板上列出了多个不同的变量匹配项和一个判定结构——要求他选出一条映射到指定路径的值通道。

解耦逐一检查了变量的内容——每一条变量的当前作用域和值与路径描述中的场景指向并不对应。它们不是一次列出的可行的多重选项——这里包含了在目标路径上执行前的多个不同场景的指定数据通道的引用目标，其中只有一条对应的路径映射数据能够在现有结构下产生合法的地址转换跳转。

他找到了映射表上标记为末端对应关系的那一组数据。数据指向的路径号和接入闸门识别门的寄存器位一致。

他按下了确认键。面板在接收到选中项之后，将他的选择传递到了闸门的锁控机构——闸门没有在连续喘振往复的开合中卡住，而是以标准的解锁流程打开了。

解耦走进闸门后的空间——不是走廊，是一间大小约十步见方的密室。密室中央的地上嵌着一台灰扑扑的终端设备，面板在待机状态下只在右上角保留了一个微小的绿色指示灯。终端上方约手掌高的位置，用光投影维持着一组悬浮结构：

"A class should have only one reason to change."

法则下方是一行翻译体："一个类只能有一个变更理由。违反者——造物内部逻辑分裂，自我冲突。"

投影旁边还附了一个连接器接口——圆口，与创世之城通用的造物诊断接口标准一致。解耦把机械兽接入接口，面板上出现了一个新的输出窗口：

"检测职责分布计数：2。建议分布层合并至标准阈值。分离规则扫描：运动驱动模块与外围环境判定模块运行在同一声明域内。潜在外围检测损耗风险：中。"

单一职责。一个类不能同时负责两件不同的事。他的造物的运动逻辑和环境扫描逻辑写在同一个结构单元中——变量在环境中相互引用，修改运动行为时不小心就会破坏扫描配置，反之亦然。物理上并不产生硬性约束限制。

他截断了机械兽当前结构中的运动驱动码和周边环境识别码之间的依存关系。运动控制独立为一个单元，只做路径规划和步态控制。扫描和判定独立为另一个单元，只处理环境数据的获取和判断。两者之间的唯一交互路径通过一个狭窄的准入接口实现：运动单元向判定单元请求"前方是否有障碍"，判定单元返回"是"或"否"，单元之间只传递这组简单的语义。

判定器重新读取了造物结构，在地面的壳体异常记录窗口中输出了一行新的状态信息：

"职责收敛：通过。变更路径隔离完成。建议保留当前结构。"

密室的一面墙在判定结束后向内打开。墙体后面是一个壁龛。壁龛中放着一本笔记——封面是深灰色的压制材料，没有书名，没有任何标识。封面正中用烫金工艺压着一个单一的几何图形：一个外侧圆环绕一个内侧点——从设计意图上可以指代多个方向的路标系统和准入节点结构。

解耦翻开笔记。

第一页靠上的位置，用一笔连贯而不潦草的字体排列出一段描述：

"单例——确保一个类只有一个实例，并提供全局访问点。适用场景：当你确定系统中某个组件最多只需要一个副本、而且该副本需要在调用层次中全局可被引用时。写法有多种，核心意思一致——把构造方法私有化，在类内部控制实例的创建，并提供一个统一的访问入口。"

笔记的第一页在末尾处画了一条横线隔断，然后另起一行写了一段更小的文字——像是作者自己的批注：

"但要小心。单例的全局可访问特性，在不同加载周期中会被多个调用流同时触及——如果没有额外的访问控制隔离，可能会出现多个调用同时在单例中竞争同一份资源的情况。"

第一页没有写完，在批注处结束了。

解耦合上笔记。他把笔记夹在外套内侧，动作不大但速度稳定。

墙外壁龛的开口在笔记被取走后没有立即关闭——闸门结构的配重使其停留在当前展开位置，在无人触碰的后续几秒中持续锁定了整个厂区所有通道的通过许可状态。

密室不再有其他出口——他沿着来路返回。前两道闸门在他接近时自动开放。

他走完前两道闸门后，回到第一条通道时，身后的管道中没有任何机构复位产生的声音——工厂在他的通行记录中为他重置了那组路径选择标记，仿佛他从未经过。但笔记实在地压在外套靠胸的位置——偏左，贴着那只仍处于待机状态的机械兽的外壳。他穿过开着的出站门回到缓冲带外侧时，身后推拉门在叠合结构中落回底座，发出一声同步啮合的低频声。

天色已经黑了。创世之城的灯光在缓冲带尽头连成一片。他把笔记内袋下方叠了一折。
