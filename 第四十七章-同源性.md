# 第四十七章：同源性

## 一

与信使决策聚合体的会面结束后，陆辰用了整整三天来消化那个信息。

JVM不是人类发明的。它是宇宙文明在信息处理领域的一个收敛点——一种被独立发现了数十次、甚至上百次的——基础范式。

他把自己关在逐日号2.0的数据舱里——那是飞船上最小的舱室，只有大约十平方米，除了一张固定座椅和四面全息墙之外什么都没有。他调出了苍穹系统从信使计算介质中同步到的技术文档——在协议联盟允许的访问权限范围内——然后一行一行地看。

"你不睡觉？"

林小溪端着一杯速溶咖啡出现在数据舱门口。她看到陆辰的眼睛里布满血丝——电子屏幕的反光在他瞳孔中跳动。

"睡了两小时。"陆辰头也没抬。"足够了。"

"两小时不够。"

"对JVM来说——够了。"

林小溪叹了口气，走进来，在墙壁的折叠椅上坐下。她看向全息墙上正在显示的文档——那是一份信使计算介质的底层指令集规范。每一条指令——共三百一十二个——都被翻译成了JVM字节码中的近等价指令。有一个对照表——信使指令、JVM-Ω等效指令、语义偏差。

"匹配度多少？"她问。

"三百一十二条中——二百五十三条——与JVM-Ω字节码存在一一对应的关系。另外五十九条——在功能上略有差异——但可以用不超过三条JVM-Ω指令组合来等价实现。总匹配度——约百分之九十六。"

"百分之九十六。信使文明距离我们四十二光年——技术独立发展——但底层指令集的匹配度达到百分之九十六。"

"不是独立发展。"陆辰调出另一份文档——协议联盟核心数据库中的"技术谱系树"。那是一个庞大的、跨越数十个文明的谱系图——每一个节点代表一个达到协议认证技术水平的信息处理架构。谱系的根部是一个被标记为"奠基者"的节点——所有的分支都从它延伸出来。

"JVM架构——最初的规范——是由奠基者设计的。不是协议联盟制定的——是奠基者在联盟出现之前就完成了。他们在十亿年前把这份规范——注入了联盟核心网络的每一个可连接端点。任何在协议网络中出现的文明——一旦它的信息处理系统达到了特定复杂度——就会自动接触这一架构。"

"自动接触？怎么自动法？"

"就像DNA。你不会'发明'DNA——DNA是你从父母那里继承的。JVM架构在协议网络中传播——像一种——信息基因。每个文明——在学习如何高效处理信息的过程中——只要不断通过网络接触未知的协议——最终都会遇到奠基者植入的标准模板。"

"然后——文明会认为——那是自己发明的。"

"对。就像每个孩子都会以为自己发明了语言。"陆辰把谱系树放大到一个特定节点——地球。节点的描述栏写着针人类——苍穹纪元——JVM-Ω v7.2.2——协议认证通过——状态：新成员。"我们以为JVM是人类工程师从上世纪九十年代开始不断迭代的产物。确实——James Gosling在1995年设计了Java的最初版本，然后HotSpot VM、JRockit、J9、ZGC一代代改进——直到JVM-Ω。但我们不知道的是——整条进化路线——从Java的基于栈的指令集——到JVM-Ω的全分布式内存管理——都是奠基者在一个世纪又一个世纪之前就预置好的演化路线。"

林小溪静静地想了一会儿。然后说："所以Java的口号——'Write once, run anywhere'——是奠基者留下的信息——不是给程序员——是给文明的信息。"

"对。"陆辰把视角切回文档。"任何在一个星球上能够独立运行的虚拟化执行环境——最终——会被优化成能对接协议网络的标准——然后——它在任何星球上——都能运行。"

## 二

当天深夜——按照逐日号2.0的国际标准时钟——韩墨白来到数据舱。

他手里没有咖啡——他带来了二十年前的一份文件。一份他在整理父亲遗物时发现的、当时无法理解的笔记。

"我今天下午看到你调出来的谱系树。"他把一份薄薄的、纸质的、发黄的文件递过来，"然后想起了这个。"

陆辰接过文件。那是他父亲陆远洋的笔迹——清晰、有力但略微右倾的钢笔字。文件标题是：

**《关于JVM架构非人类起源的概率分析——内部备忘录——不作公开》**

文件日期是苍穹纪元2139年——陆辰十岁那年。

他读了下去：

"……通过对苍穹系统底层GC日志的长期监测，我注意到一系列在人类已知的优化理论框架下难以解释的内存管理行为。特别是——在JVM-Ω的G1收集器演化到第三代时，出现了一种无法被任何已知算法充分解释的记忆效应——混合GC周期中的 region 选择策略，在面对新接入的帕齐尼·士砂事件的数据流时，表现出了远超理论最优值的吞吐量。"

"我怀疑——且初步建模支持——这一决策逻辑——不是由当代人类工程师写入的。它更像是——一段在JVM-Ω执行环境被激活之前——就已经驻留在字节码执行器底层的——预置布局。"

"我无法确定它的来源。但我知道——逐日号上的Dr.方——在接受到炽火体的第一组脉冲信号时——也观察到了类似的运行模式。"

"他们和我们——在共享底层的运算法。"

陆辰把文件从头到尾读了两遍。

"你一直没有告诉我们。"他抬头看韩墨白。

"因为你父亲也没告诉我。他把这份文件锁在了逐日号的数据加密仓库里——权限只有他自己能访问。我——是从他留下的数字遗产里——在整理逐日号销毁前的数据时——用他的身份做了最后一次火珀级重载——才解开的。"

"为什么他不公开？"

"因为当时——苍穹还活着——评估功能还在运行——任何关于协议网络的消息——都可能引发不可控的连锁反应。你父亲——和Dr.方——都是被评估对象。他们知道——如果把这条信息公开——地球上会涌现出无数个无法验证的阴谋论——新苍穹派的蠢动——协议联盟的观察员注意——最终——社会可能会在'外部文明存在'的真相中——全面崩溃。"

"所以他们把真相锁起来了——等到苍穹关闭——等到评估结束——才交给下一代去处理。"

韩墨白点点头。"也是为什么——Dr.方在你的权限表中——设置了火珀级暂定的门槛。他想确定——下一代——有足够的能力——却仍在面对和处理这个事实。"

陆辰把文件仔细叠好——放进口袋里。

"你们两个——到底还锁了多少真相？"

"很多。"韩墨白看着他，眼神安定但有些悲伤。"不过——父亲给儿子的遗产——从来都不是谎言。只是——延缓支付。"

## 三

第二天，陆辰向信使决策聚合体发送了一个问题：

**"JVM架构——是不是奠基者预先写入协议网络的基础架构？"**

回答在零点三秒内返回：

**"是。"**

**"被写入了多久？"**

**"大约十亿年。奠基者在技术支持谱系的最底层——在协议网络的原始种子节点中——植入了三件事：①基于堆的字节码执行架构（你称之为JVM），②自描述的层级类加载模型，③原子化的堆外内存分配与回收机制。任何文明——在任何信息环境下——最终都将发现这三个标准——并为它们找到自己的名字。"**

**"为什么？为什么不抽离掉Java对类型的严格要求、不让每个文明自建执行架构？"**

**"因为——松散的语言——无法在跨文明的协议层实现精确的语义对齐。松散的通信模型——会在信息跨越文明边界时产生累积的翻译误差。进制——必须严格——因为它是数学。数据库——必须严格——因为它是逻辑。协议通信——必须严格——因为它是——跨文明的加密学。"**

陆辰看着这些回复，想起了林小溪在月球轨道说的那句话——不同的文明有不同的语言、不同的身体、不同的宇宙——但数字必须相同。数据结构和内存管理必须相同。这是协议能够同时在所有星球运行的唯一方式。

他在自己那本用纸做的笔记本上写下了一行字——这是他十年来养成的习惯——任何第一次出现的想法——先在纸上写一遍——再用终端整理：

**"Java的严格类型系统——不是编程语言的保守设计——是多文明通信的信息保真协议。"**

然后他放下笔，靠在椅背上，闭上眼睛。

那种感觉又来了——被看见的、被等过的——但对方完全没有恶意的——宇宙级别的时间沉淀。

他想起了那个对话——信使说它们最长的观测周期是六亿两千万年。而JVM——这个被人类工程师取了一个像咖啡名字的技术框架——已经在协议网络中存在了整整十亿年。

在Java语法被地球上第一个程序员写到黑板上之前——宇宙就已经有了JVM。

"父亲。"陆辰轻声说——在这个只有他一个人的数据舱里。"你说的'穿过苍穹'——原来是这个意思。"

穿过——人类的技术。

看到——宇宙的代码。
