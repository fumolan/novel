# 第1章：苍穹之下

## 一

2157年。苍穹纪元第四十二年。

陆辰盯着屏幕上的堆内存曲线，已经有四个小时没有离开过座位。

曲线像一条缓慢抬头的蛇——不陡，但一直在涨。从三天前的47%占用率，涨到今早的68%，然后午餐时突破了74%。现在是下午两点四十分，数字跳到了79%。

不正常。

苍穹系统的堆内存总容量是EB级别——地球十二座超级数据中心的全部算力聚合在一起。占用率正常情况下应该在40%到60%之间浮动，随着城市群的活动周期自然起伏。79%不在正常范围内。

陆辰调出了堆内存的详细分布图。

老年代稳定——大约占55%，符合预期。

新生代偏高——Eden区有大量未被回收的短期对象。这些短期对象本应在Eden区填满后被Minor GC清空，但它们没有被清空。

或者——GC清空了它们，但新的对象分配速度超过了回收速度。

陆辰把鼠标指针移到新生代的实时活动图上。Eden区的占用率曲线在过去的七十二小时内呈现出一种锯齿形——陡升→平缓→陡升→平缓。每个"陡升"的间隔大约四十分钟。四十分钟一次大对象分配。

什么东西在每四十分钟分配一次大对象？

他用命令行工具连接了苍穹气候模块的JMX端口，开始查询堆内存中的对象分布：

```
jmap -histo:live 7841
```

数据流回。排在第一位的类名他没有立刻认出——一个名字长度约三百个字符的匿名内部类，包名指向苍穹气候模块的温带子系统。实例数量：七个。每个实例占用的堆内存：相当于一个中等城市一天的气候运算量。

七个实例。六个应该是正常的——温带子系统有六个分区：西欧、东亚、北美、南美、非洲、澳洲。第七个分区是不在官方架构图中的。

陆辰调出了类加载器清单，查找这个匿名内部类的来源。加载器显示为"系统默认加载器"，类文件路径指向苍穹核心代码库，最后编译时间戳：十年前。

十年前的代码——在今天开始疯狂分配内存。

"陆辰。"

身后有人叫他。陆辰没有回头——他的眼睛还盯在那条堆内存曲线上。

"你看了四个小时了。要不要换人？"

说话的是他的搭档老张，底层维护组的老人，在苍穹干了十五年。

"不用。"陆辰说，"我快找到原因了。"

"每次你都这么说。"老张把一杯已经凉了的茶放在他桌上，"上次你说'快找到原因了'，我在机房等了你十二个小时。"

"这次是真的。"

老张走开了。陆辰继续操作。他需要看堆内存中那个匿名内部类的实例详情。更详细的对象分布，更精确的引用链。

他生成了一个堆转储文件。

文件大小：17GB。下载时间：大约四分钟。

四分钟后，他用MAT打开了堆转储。引用链分析显示——那七个实例被一个静态集合引用着。集合本身属于一个被标记为"弃用"的模块。弃用模块中的静态集合，引用着仍在活跃分配内存的内部类实例。

从弃用模块到活跃实例，有一段GC Roots不可达的路径——逻辑上它们应该被回收。但实际上它们没有被回收。因为那个静态集合被一个线程持有——而线程的启动配置文件中，类层级归属存在一个额外的路径注册条目——在任何宿主服务器的进程列表和子系统代码目录中均不存在对应的物理文件。

"幽灵模块。"陆辰自言自语。

他把鼠标指针移到线程分析面板上，找到持有该静态集合的线程。线程名称是默认生成格式——Thread-XXXX。但线程的启动时间戳引起了陆辰的注意：十年零三个月前。

和那个匿名内部类的编译时间一致。

十年前创建的一个线程，持有一个十年前的弃用模块中的静态集合，引用着十年前编译的内部类实例——这些实例在今天开始疯狂分配内存。

陆辰打开了该线程的栈追踪。

追踪从栈顶到栈底一共四十七层。他逐层往下翻。在栈底的倒数第三行——一个if分支中——他看到了一行混在null指针判空语句旁注释。

注释内容只有几个字符：

`// J-1172`

陆辰的手指在键盘上停住了。

J-1172。

他六岁那年的某个深夜。父亲坐在工作室里调试一段代码，他蹲在旁边看。父亲指着屏幕右上角一个闪烁的指示灯说："看到这个编号了吗？这是爸爸的工号。以后你在系统里找到这个数字——就知道爸爸还在。"

J-1172。他的父亲的工号。

"陆辰？"

老张的声音从远处传来。陆辰没有回答。他把那行注释放大了两倍，截图，保存。

然后他开始往下刨——从那行注释所在的位置继续向下追。注释在一个if(false)分支中。死分支。不被执行的代码。但注释留在了那里。

他追查了这段代码的版本历史。Git日志显示：十年前，一次commit，提交者工号J-1172。commit信息只有一个词："种子。"

种子。

commit的文件只有一个：一个类文件。类名被混淆过，无法直接阅读。但class文件的结构是标准格式——他可以用字节码反编译工具打开它。

他打开了字节码。

反编译后的代码不长——大约八十行。主要逻辑是从一个静态资源中读取一个字符串，然后把这个字符串写入苍穹系统的内部日志通道。

字符串的内容，在他反编译完最后一个魔数后露了出来。

"儿子，如果你看到这行字，说明堆快要溢出了。时间不多了。"

## 二

陆辰没有把这个发现上报。

他不知道怎么解释。一个十年前的commit，一个if(false)的死分支，一行只有他能看懂的注释，一条写入日志通道的字符串消息。如果上报，流程会是这样：他提交异常报告→分析组确认→安全组介入→调查J-1172工号的原始提交人。调查结果：J-1172，十年前被标记为"失踪"。

然后呢？一个"失踪"的人留下了代码——过了十年才在执行中触发一个假条件分支，向苍穹系统的核心日志通道写入了一份数据载荷——这个行为的窗口刚好在今天被陆辰抓到了。

这听起来像他在编故事。

他没有上报。他在MAT中继续分析那十七条引用链。弃用模块→静态集合→持有线程→匿名内部类→堆内存溢出。每一条引用链的末端都连着一个完整的字符串对象——内容不是气候数据。是文本。加密的文本。

十年前的某个时候，他的父亲把某种加密数据编码成了气候模块可以处理的例行分配序列，通过一个被标记为"弃用"的模块的静态集合引用路径逐段写进老年代不会被一次回收清理掉的位置——然后用一个延迟十年的信号告诉陆辰"我在这里"。

他关掉了MAT。然后站起来，走到窗边。

苍穹的防护网在几百米高的地方蜿蜒铺展，像一层半透明的琥珀，把整个天空笼在其中。透过它可以隐约看到真正天空的颜色——但只是隐约。新上海的楼群在防护网的映照下泛着一种均匀的、不真实的白光。

他在通讯终端上打了一行字。收件人：苍穹异常事件处理中心。内容是工单模板的标准用语——"堆内存占用率异常，已定位至气候模块弃用子系统的静态集合引用泄漏。建议对温带分区六个子系统做一次GC触发维护。"

他没有提到那行注释。没有提到J-1172。没有提到父亲的名字。

工单提交后几秒，自动回复抵达：已受理，排序中。

从"受理"到"排序中"到"指派责任人"到"真正有人去看"——在苍穹体系中通常需要三到五天。十条提交有一半会在第三天被自动标记"待进一步确认"。陆辰的工单大概也会走同样的路。

他关了终端。窗外，苍穹防护网的表面有一块区域的亮度在轻微波动——不是错觉，是异常区域发生了微小的偏移，又在眨眼间复位。

他盯着那片区域看了几秒。

堆快要溢出了。那是父亲的消息。

时间不多了。

他低头看着自己的手指——六岁时父亲教他用键盘，说："以后你要是用这套东西解了一个没人解得出来的bug——那就是爸爸最高兴的一天。"他不知道今天算不算"解了一个无人解决的bug"。窗口期已在核心日志内部，搭载了一份延迟了十年的数据的payload。他不知道payload之中最终会露出什么。

但他知道一件事。

J-1172还在苍穹中。

不是作为人类在某个遥远的地方——是作为一个工号，一个数字签名，一行混在if分支里的注释。

他在苍穹深处。

而他正在召唤陆辰。
